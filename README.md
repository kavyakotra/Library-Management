Library Management System

Overview

The Library Management System is a SQL-based database project designed to manage books, authors, members, and book issue/return records. It demonstrates the use of SQL concepts such as table creation, relationships, joins, constraints, and queries.

Features

- Manage book details
- Store author information
- Register library members
- Issue and return books
- Track available book quantity
- Generate reports using SQL queries

Technologies Used

- MySQL
- MySQL Workbench

Database Tables

- Authors
- Books
- Members
- BookIssue
- BookReturn

SQL Concepts Used

- CREATE TABLE
- INSERT, UPDATE, DELETE
- SELECT Queries
- INNER JOIN
- LEFT JOIN
- RIGHT JOIN
- FULL JOIN (using UNION)
- WHERE, ORDER BY, GROUP BY
- Aggregate Functions
- Primary Keys and Foreign Keys

Project Objectives

- Design a relational database for a library.
- Maintain book and member records efficiently.
- Practice real-world SQL queries.
- Improve database management skills.

Future Enhancements

- Add fine calculation for late returns.
- Implement login and user roles.
- Create a graphical user interface.
- Add stored procedures and triggers.
